大概就是一个用python写的又烂又不安全的云存储服务端与客户端吧
基于原生TCP Socket 实现的轻量跨设备云存储工具，支持文件上传/下载/列表查看/远程删除，密码SHA256哈希鉴权，带传输进度与速度显示，多客户端并发连接。

## 项目文件说明
```
Cloud-Storage-TCP/
├── Server-IPv6.py       # 云存储服务端（部署在存放文件的设备）
├── User-IPv6.py         # 云存储客户端（任意设备连接服务端）
├── pwd_config.json # 服务端自动生成，存储访问密码
├── server_storage/ # 服务端自动生成，存放所有云端文件
└── client_download/# 客户端自动生成，存放下载到本地的文件
```

## 功能特性
### 服务端
1. 多线程并发，支持同时多个客户端连接
2. SHA256哈希密码鉴权，原始密码不通过网络传输
3. 防止路径穿越攻击，非法文件名直接拦截
4. 单文件上传上限10GB，磁盘剩余空间预警
5. 控制台支持实时修改访问密码
6. 自动统计磁盘总容量、已用、剩余、存储目录占用
7. TCP消息分隔符解决粘包问题，传输稳定

### 客户端
1. 交互式命令行终端，操作简单
2. 文件上传/下载实时进度条 + 平均传输速度
3. 彩色文字输出，区分成功/警告/错误信息
4. 本地自动创建下载文件夹
5. 指令帮助文档，支持随时查看
6. 断开连接自动回收socket资源

### 支持指令（客户端内执行）
| 指令 | 功能说明 |
|------|--------|
| list | 查询服务器所有文件 + 磁盘存储状态 |
| upload 本地文件路径 | 将本地文件上传至服务端 |
| download 文件名 | 从服务端下载文件到本地client_download |
| del 文件名 | 删除服务端指定文件 |
| help | 查看全部可用指令 |
| exit | 断开服务器连接，退出客户端 |

## 快速部署&使用教程
### 1. 部署服务端（存放文件的电脑/服务器）
1. 修改 `Server-IPv6.py` 配置（按需）
   ```python
   HOST = ""       # 允许局域网/公网访问
   PORT = 40000            # 通信端口，客户端必须保持一致
   MAX_UPLOAD_SIZE = 10 * 1024 * 1024 * 1024 # 最大单文件10GB
   WARN_FREE_SPACE = 10 * 1024 * 1024 * 1024 # 剩余10GB弹窗预警
   ```
2. 运行服务端
   ```bash
   python Server-IPv6.py
   ```
3. 服务端控制台操作
   - `setpwd`：修改云存储访问密码（修改后需重启服务端生效）
   - `start`：启动监听，接受客户端连接
4. 首次运行自动生成：
   - `pwd_config.json`：存储密码，默认密码 `Cloud@2026`
   - `server_storage`：云端文件存储目录

> 局域网使用：服务端防火墙放行40000端口；公网使用需端口映射。

### 2. 启动客户端（任意访问设备）
1. 修改 `User-IPv6.py` 连接配置
   ```python
   SERVER_IP = "省略"  # 填写服务端IP地址
   SERVER_PORT = 40000           # 和服务端端口保持一致
   ```
2. 运行客户端
   ```bash
   python User-IPv6.py
   ```
3. 输入服务端设置的访问密码，鉴权通过即可进入交互终端

## 安全说明
1. 密码不会明文传输：客户端将密码做SHA256哈希后发送，服务端仅比对哈希值
2. 严格文件名过滤，拦截 `../`、绝对路径，防止路径穿越漏洞
3. 单文件上传大小限制，避免磁盘被恶意占满
4. 错误指令、非法操作统一返回提示，无敏感报错泄露服务器信息
5. 连接断开自动关闭socket，不会残留无效连接
6. 本程序不含ssl加密，可选择自行修改代码以实现，因此造成的隐私泄露不由GameFlant承担

## 常见问题
### 1. 客户端连接失败
- 检查服务端是否执行 `start` 启动监听
- 核对 `SERVER_IP`、`PORT` 两端完全一致
- 关闭双方防火墙/放行40000端口
- 局域网确保两台设备在同一WiFi/网段
- 公网使用时请确保服务端对应端口放行/路由器或光猫放行正常

### 2. 密码验证失败
- 服务端修改密码后必须重启 `Server-IPv6.py` 才会生效
- 输入密码无空格，区分大小写
- 删除 `pwd_config.json` 重启服务端恢复默认密码 `Cloud@2026`

### 3. 上传提示磁盘不足
- 服务端磁盘剩余空间小于待上传文件大小，清理文件后重试

### 4. 文件传输中断
- 大文件传输等待进度条走完，不要强制关闭程序
- 网络波动会断开连接，重新登录再次操作即可
